# Warehouse Management System – Database Design Project

> SQL, Data Modeling, Business rule mapping

---

## Context 
- **Objective:** Design and implement a relational database system to support:
  - Inventory management  
  - Procurement (supplier → warehouse)  
  - Distribution (warehouse → agents)  

---

## Steps
- Analyzed warehouse operations including procurement, storage, and distribution
- Translated business workflows into structured system logic
- Designed **Entity-Relationship Diagram (ERD)** to define entities, relationships, and cardinality  
- Converted ERD into **Relational Database (RD) schema**  
- Applied **normalization (up to [3NF / BCNF – fill])** to eliminate redundancy  
- Built SQL scripts for:
  - **Schema creation** (CREATE TABLE)  
  - **Data population** (INSERT INTO)  
  - **Constraint enforcement** (PK, FK, NOT NULL, CHECK, etc.)
